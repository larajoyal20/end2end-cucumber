# End-to-End Testing Framework

A comprehensive end-to-end testing automation framework using **Cucumber BDD** and **Playwright** with TypeScript. This project automates an e-commerce order placement workflow with detailed Allure reporting.

## 🎯 Project Overview

This project implements behavior-driven development (BDD) testing for an e-commerce application. It covers the complete user journey from login through product selection to order payment and confirmation.

## 🛠️ Tech Stack

- **Test Framework**: Cucumber.js (BDD)
- **Browser Automation**: Playwright
- **Language**: TypeScript
- **Test Reporting**: Allure, Cucumber HTML Reporter
- **Node.js**: JavaScript Runtime

## 🚀 Getting Started

### Prerequisites

- **Node.js** (v18+)
- **npm** (v9+)
- Modern browser (Chrome/Chromium for testing)

### Installation

1. Clone or navigate to the project directory:
   ```bash
   cd d:\end2end-cucumber
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

## 🧪 Running Tests

### Run Tests Only
```bash
npm run test
```
Executes all Cucumber scenarios and generates JSON reports in the `reports/` directory and Allure result files in `allure-results/`.

### Generate HTML Report
```bash
npm run report:html
```
Creates an HTML report using Cucumber HTML Reporter.

### Generate Allure Report
```bash
npm run report:allure
```
Serves an interactive Allure report in your browser.

### Run Tests + Generate All Reports (Complete Workflow)
```bash
npm run test:report
```
Executes tests and generates both HTML and Allure reports.

## 📊 Reports

### Cucumber HTML Report
- **Location**: `reports/cucumber-report.html`
- **Generated by**: cucumber-html-reporter
- **Contains**: Test pass/fail status, execution duration, step details

### Allure Report
- **Location**: `allure-report/`
- **Generated by**: allure-cucumberjs
- **Features**:
  - Interactive test dashboard
  - Timeline view
  - Test categorization
  - Defect tracking
  - Trend analysis

### Test Results Data
- **JSON Results**: `reports/cucumber-report.json`
- **Allure Results**: `allure-results/` (contains container and result JSON files)
- **Screenshots**: `screenshots/` (captured on test failures)

## ⚙️ Configuration Files

### cucumber.js
Configures Cucumber test runner:
- Step definitions path: `steps_defination/**/*.ts`
- Support files path: `support/**/*.ts`
- Feature files path: `features/**/*.feature`
- Output formats: Progress + Allure reporting

### playwright.config.js
Configures Playwright browser automation:
- Browser: Chromium (Desktop Chrome)
- **Headless Mode**: Disabled (headless: false) - runs in visible browser
- **Retries**: 0 (local) | 2 (CI)
- **Trace**: Recorded on first retry
- **Parallel Execution**: Enabled

## 📝 Best Practices

1. **Page Object Model**: Uses POM pattern for maintaining page selectors
2. **BDD Approach**: Tests written in Gherkin for business readability
3. **Separation of Concerns**: Steps, pages, and support logic separated
4. **Screenshot Capture**: Automatic screenshots on failures
5. **Detailed Reporting**: Multiple report formats for different stakeholders

## 🐛 Troubleshooting

**Tests Failing?**
1. Check internet connection (tests interact with live application)
2. Verify credentials in `endtoEnd.feature` are still valid
3. Check if target application is accessible
4. Review screenshot in `screenshots/` to see failure point
5. Check Allure report for detailed step logs

**Report Generation Issues?**
1. Ensure `allure-commandline` is installed: `npm install allure-commandline`
2. Check that test results exist in `allure-results/`
3. Run `npm run report:html` before `npm run report:allure`
